# 国风轻交互文史探索网页应用

## 项目简介

本项目是一款轻量化、纯静态的国风文史探索交互网页应用。

- 使用原生 HTML、CSS、JavaScript 开发
- 无需后端环境，零依赖运行
- 支持本地运行和静态部署
- 以宋代文人苏轼、李清照为核心内容，通过地图探索、诗词互动、收藏解锁等方式提供沉浸式文史体验

用户可以跟随文人足迹游历四城（济南、黄州、杭州、金华），阅读文史故事，完成诗词答题，解锁收藏内容，最终生成属于自己的游历长卷。

## 产品特色

### 1. 水墨地图探索

首页展示宋代风格水墨地图，四城光点按地理位置分布。点击城市光点查看古迹列表，已解锁城市显示墨色并带扩散动画，未解锁城市显示灰色。

### 2. 文史内容探索

涵盖四座历史文化名城，每城三处代表性古迹，共12处：

- **济南**：漱玉泉、大明湖、趵突泉（李清照主题）
- **黄州**：东坡赤壁、定慧院、雪堂（苏轼主题）
- **杭州**：西湖、望湖楼、孤山（苏轼主题）
- **金华**：八咏楼、双溪、金华寓所（李清照主题）

每处古迹配有文人创作背景故事和经典诗词。

### 3. 诗词答题与解锁机制

每处古迹对应一道诗词填空题，用户输入答案提交：

- 支持宽松比对（去除空格、标点差异）
- 答对自动解锁该古迹，同步保存到 localStorage
- 答错显示正确答案，可重新答题
- 城市3处古迹全部解锁后，触发城市达成祝贺

### 4. 藏书阁收藏

按城市分栏展示已解锁的诗词卡片：

- 四列布局，每列显示城市名称和收集进度（X/3）
- 已解锁显示完整诗词（标题+作者+全文）
- 未解锁显示灰色占位
- 城市达成后显示专属水墨边框和"✦ 达成"徽章

### 5. 今日诗签

首页侧边栏随机展示一句经典诗句：

- 包含13句苏轼和李清照的经典诗句
- 点击"抽取诗文"按钮随机切换
- 诗文抽签页面提供更大尺寸展示

### 6. 风物画集

展示已解锁的古迹插画和城市全景图：

- 按城市分栏，每城3处古迹+1张全景（城市全解锁后显示）
- 已绑定图片资源的古迹显示真实插画
- 点击插画查看大图预览，支持下载图片

### 7. 游历长卷

基于 Canvas 绘制个人探索记录：

- 横向卷轴展示已解锁内容，可左右滑动查看
- 包含标题、城市分卷、古迹插画、诗词、印章
- 支持选择城市专属边框（城市达成后解锁）
- 支持添加篆刻印章（全局通关后解锁）
- 支持下载 PNG 长图保存

    │   ├── huangzhou/      # 黄州古迹插画+全景图

    │   └── jinhua/         # 金华古迹插画

    └── stamps/             # 印章图片（hangzhou.png, huangzhou.png, jinan.png, jinhua.png）

### 目录说明

**assets/**

用于存放视觉资源：

- `gallery/`：按城市分类的古迹插画和城市全景图
- `map/`：水墨风格地图背景
- `scroll/`：长卷背景图片
- `stamps/`：城市印章图片

**data.js**

预留的数据文件，当前未使用。设计上用于未来将内容数据（城市、古迹、诗词）从 `script.js` 中分离出来，实现内容数据和页面逻辑的解耦，便于内容更新和维护。

**script.js**

包含项目核心逻辑：

- 数据层：城市、古迹、诗词、印章、边框配置
- 存档层：localStorage 读写、解锁状态管理
- 导航层：视图切换
- 地图层：光点渲染、点击交互
- 弹窗层：古迹列表、答题、成就祝贺
- 答题层：答案判分、解锁逻辑
- 藏书阁、风物画集、长卷渲染

## 技术实现

### 技术栈

- **HTML5**：语义化页面结构
- **CSS3**：色彩变量、布局、动画、水墨效果
- **JavaScript ES2020**：模块化设计，约1900行
- **localStorage**：存储解锁状态、长卷配置、全局通关标记
- **Canvas API**：游历长卷绘制、插画生成、印章渲染

### 架构设计

项目采用纯前端静态架构：

- **零后端依赖**：无需服务器，只需静态文件托管
- **数据本地存储**：所有用户数据保存在浏览器本地
- **离线可用**：下载页面后可离线运行
- **跨平台兼容**：支持桌面和移动设备

## AI辅助开发流程

本项目在开发过程中广泛使用 AI 工具辅助完成，以下是主要流程：

### 需求设计

AI 辅助分析产品方向，拆解功能模块，设计用户流程：

- 将产品描述转化为技术需求
- 划分独立功能模块（地图探索、答题系统、收藏系统、画集展示、长卷生成）
- 梳理用户从进入到完成的完整体验路径
- 识别潜在技术难点（Canvas 异步图片加载、localStorage 状态管理）

### 视觉设计

AI 辅助生成国风视觉素材：

- **水墨地图背景**：宋代水墨风格，淡墨山水轮廓，宣纸质感
- **古迹插画**：按城市和古迹分类生成水墨风格插画
- **长卷背景**：宋代卷轴展开样式，边缘淡墨晕染
- **印章图片**：中国传统篆刻印章风格

**Prompt 设计思路**：明确时代风格（宋代）、艺术风格（水墨）、质感（宣纸）、场景描述、构图要求。

### 编程开发

AI 辅助完成代码实现：

- **HTML结构**：生成页面骨架、导航、弹窗模板、视图容器
- **CSS样式**：定义色彩变量、布局、动画、水墨效果
- **JavaScript功能**：
  - 数据层：定义城市、古迹、诗词数据结构
  - 地图模块：编写 `renderMap()` 函数处理光点渲染和点击事件
  - 答题模块：编写 `checkAnswer()` 函数实现答案判分和解锁逻辑
  - Canvas长卷：编写 `renderScrollCanvas()`、`drawSiteIllustration()` 等绘制函数

### Debug优化

AI 帮助分析和解决开发过程中的问题：

1. **Canvas长卷背景显示问题**：定位多层CSS覆盖导致背景图片被遮挡，提供最小修改方案
2. **风物画集图片为空问题**：发现变量名冲突（`isUnlocked` 同时作为函数名和变量名），修复循环中断问题
3. **Canvas古迹图片加载问题**：解决同步渲染与异步加载的时序问题，实现"缓存+重绘"策略
4. **高清Canvas适配**：添加 `devicePixelRatio` 适配，解决文字模糊问题
